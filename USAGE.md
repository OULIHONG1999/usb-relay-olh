# USB Relay 使用指南

## 📱 快速开始

### 1. 启动Android服务端

```bash
# 在Android设备上
1. 打开"USB Relay"应用
2. 点击"Start Server"
3. 记下显示的IP地址（例如: 192.168.31.97）
```

### 2. 连接USB设备

```bash
# 通过OTG线将USB设备连接到Android
# 应用会自动检测并显示设备列表
```

### 3. 启动Windows客户端

```bash
cd relay-client
python win_gui.py
```

### 4. 连接到服务器

```
1. 输入Android设备的IP地址
2. 端口保持3240
3. 点击"连接"按钮
```

## 🎯 主要功能

### ✅ 设备发现
- 自动检测Android连接的USB设备
- 实时显示设备信息（VID/PID/名称）
- 设备插拔自动更新

### ✅ 日志广播
- 接收服务器推送的日志
- 结构化显示（时间戳/级别/消息）
- 断开ADB后仍能查看

### ✅ 设备导入（框架）
- 选择设备并请求导入
- 接收导入状态响应
- ⚠️ 当前为占位符实现

## 📊 界面说明

### 连接设置区域
- **服务器地址**: Android设备的IP
- **端口**: 默认3240
- **连接按钮**: 连接/断开
- **状态标签**: 显示连接状态

### 设备列表区域
- **ID**: 设备编号
- **VID**: Vendor ID
- **PID**: Product ID  
- **Class**: 设备类别
- **Name**: 设备名称

### 操作按钮
- **刷新设备**: 手动刷新列表
- **导入选中**: 导入选中的设备
- **Ping**: 测试连接

### 日志区域
- 显示所有通信日志
- 服务器推送的日志
- 本地操作记录

## ⚠️ 注意事项

### 网络要求
- Android和Windows必须在同一网络
- 确保防火墙允许3240端口
- WiFi连接更稳定

### USB权限
- 首次连接USB设备需要授权
- 勾选"默认用于此USB设备"
- 拒绝授权会导致设备不可用

### 版本信息
- App版本号显示在Server Control卡片
- 格式: Version: X.Y.Z
- 每次构建自动递增

## ❓ 常见问题

### Q1: 连接失败？
**检查**:
- IP地址是否正确
- 服务器是否已启动
- 防火墙是否阻止
- 网络是否通畅

**解决**:
```bash
# 测试网络连接
ping <android-ip>

# 检查端口
telnet <android-ip> 3240
```

### Q2: 设备列表为空？
**可能原因**:
- 没有USB设备连接
- USB权限未授予
- 设备不兼容

**解决**:
1. 确认USB设备已物理连接
2. 检查Android通知栏的USB授权
3. 点击"Refresh Devices"

### Q3: 导入设备失败？
**当前状态**: 
- 导入功能是框架实现
- 返回成功但提示需要驱动集成
- 无法实际使用USB设备

**未来计划**:
- 整合usbip-win2驱动
- 实现完整的USB虚拟化
- 让Windows能真正使用USB设备

### Q4: 日志显示乱码？
**原因**: 编码问题
**解决**: 确保UTF-8编码

### Q5: App闪退？
**已修复**: 
- 协议头部格式问题已解决
- 空设备列表处理已完善
- 如果仍闪退，请报告bug
