# 版本号管理使用指南

## 📋 概述

项目已实现自动版本号管理系统，每次构建App时会自动读取并显示版本号。

## 🎯 功能特性

1. ✅ **版本号配置文件** - `version.properties`独立管理版本
2. ✅ **自动版本递增** - PowerShell脚本自动更新版本号
3. ✅ **UI界面显示** - App界面上实时显示当前版本
4. ✅ **一键构建** - 自动更新版本并构建安装

## 📁 文件说明

### 1. version.properties
**位置**: `relay-host/USBRelay/version.properties`

**格式**:
```properties
VERSION_MAJOR=1
VERSION_MINOR=0
VERSION_PATCH=2
```

**说明**:
- `VERSION_MAJOR`: 主版本号（重大更新）
- `VERSION_MINOR`: 次版本号（新功能）
- `VERSION_PATCH`: 修订号（bug修复）

### 2. update_version.ps1
**位置**: `relay-host/USBRelay/update_version.ps1`

**功能**: 自动更新版本号

**用法**:
```powershell
# 更新patch版本 (1.0.2 -> 1.0.3)
.\update_version.ps1 patch

# 更新minor版本 (1.0.2 -> 1.1.0)
.\update_version.ps1 minor

# 更新major版本 (1.0.2 -> 2.0.0)
.\update_version.ps1 major
```

### 3. build_with_version.bat
**位置**: `build_with_version.bat` (项目根目录)

**功能**: 一键更新版本、构建、安装

**用法**:
```batch
# 在项目根目录运行
build_with_version.bat
```

**流程**:
1. 自动更新patch版本号
2. 清理旧构建文件
3. 编译Android应用
4. 询问是否安装到设备

## 🚀 使用方法

### 方法1: 手动更新版本（推荐用于开发）

```powershell
# 1. 进入Android项目目录
cd relay-host\USBRelay

# 2. 更新版本号（选择类型）
.\update_version.ps1 patch    # bug修复
.\update_version.ps1 minor    # 新功能
.\update_version.ps1 major    # 重大更新

# 3. 构建应用
.\gradlew.bat assembleDebug

# 4. 安装到设备
adb install -r app\build\outputs\apk\debug\app-debug.apk
```

### 方法2: 一键构建（推荐用于快速测试）

```batch
# 在项目根目录直接运行
build_with_version.bat
```

脚本会自动：
- 更新patch版本号
- 清理并重新构建
- 询问是否安装到设备

### 方法3: 不更新版本直接构建

```powershell
cd relay-host\USBRelay
.\gradlew.bat assembleDebug
```

这种方式会使用`version.properties`中的当前版本号，不会自动递增。

## 🔧 技术实现

### build.gradle.kts 配置

```kotlin
// 读取version.properties
val versionPropsFile = file("../version.properties")
var versionMajor = 1
var versionMinor = 0
var versionPatch = 0

if (versionPropsFile.exists()) {
    val props = Properties()
    FileInputStream(versionPropsFile).use { props.load(it) }
    versionMajor = props.getProperty("VERSION_MAJOR", "1").toInt()
    versionMinor = props.getProperty("VERSION_MINOR", "0").toInt()
    versionPatch = props.getProperty("VERSION_PATCH", "0").toInt()
}

val versionName = "$versionMajor.$versionMinor.$versionPatch"
val versionCode = versionMajor * 10000 + versionMinor * 100 + versionPatch

android {
    defaultConfig {
        versionCode = versionCode
        versionName = versionName
        
        // 生成BuildConfig常量
        buildConfigField("int", "VERSION_CODE", "${versionCode}")
        buildConfigField("String", "VERSION_NAME", "\"$versionName\"")
    }
    
    buildFeatures {
        buildConfig = true  // 启用BuildConfig
    }
}
```

### MainActivity.kt 显示

```kotlin
Text(

)
```

## 📊 版本号规则

### Semantic Versioning (语义化版本)

格式: `MAJOR.MINOR.PATCH`

**何时更新各版本号**:

- **MAJOR (主版本)**: 
  - 不兼容的API变更
  - 架构重构
  - 重大功能改版
  
- **MINOR (次版本)**:
  - 向后兼容的新功能
  - 新设备支持
  - UI改进
  
- **PATCH (修订号)**:
  - Bug修复
  - 性能优化
  - 小的改进

### versionCode 计算规则

```
versionCode = MAJOR * 10000 + MINOR * 100 + PATCH
```

**示例**:
- 1.0.2 → 10002
- 1.2.5 → 10205
- 2.0.0 → 20000

这样可以确保版本号递增时，versionCode也严格递增，符合Google Play要求。

## ❓ 常见问题

### Q1: 如何查看当前版本？
**A**: 
- 方法1: 打开`version.properties`文件
- 方法2: 运行`.\update_version.ps1`（会显示当前版本）
- 方法3: 打开App，在Server Control卡片中查看

### Q2: 版本号没有更新？
**A**: 
- 检查是否正确运行了`update_version.ps1`
- 确认`version.properties`文件存在且可写
- 清理构建缓存：`.\gradlew.bat clean`

### Q3: 能否自定义版本号？
**A**: 
可以直接编辑`version.properties`文件：
```properties
VERSION_MAJOR=2
VERSION_MINOR=1
VERSION_PATCH=0
```

### Q4: BuildConfig.VERSION_NAME为空？
**A**: 
- 确保`build.gradle.kts`中启用了`buildConfig = true`
- 清理并重新构建：`.\gradlew.bat clean assembleDebug`

### Q5: 如何在代码中使用版本号？
**A**: 
```kotlin
// 在任何Activity或Fragment中
val version = BuildConfig.VERSION_NAME  // "1.0.2"
val code = BuildConfig.VERSION_CODE    // 10002
```
