## 发现的问题

### 🔴 严重问题

#### 1. 协议头部格式不一致
**位置**: win_gui.py line 207, 309  
**问题**: struct格式字符串从`HIHHI`(14字节)改为`HIHII`(16字节)  
**状态**: ✅ 已修复  
**验证**: 需要测试连接稳定性

#### 2. App闪退风险
**位置**: UsbService.kt sendDeviceListUpdate()  
**问题**: 如果availableDevices为空，可能发送空JSON数组  
**影响**: 客户端可能解析失败  
**建议**: 添加空检查或发送默认响应  
**状态**: ⚠️ 待验证

### 🟡 中等问题

#### 3. 设备ID映射问题
**位置**: UsbService.kt handleImportDevice() line 416  
**问题**: 使用`elementAtOrNull(devId - 1)`假设设备ID是连续的索引  
**风险**: 如果设备被移除，ID可能不连续  
**建议**: 使用Map直接查找，或者维护ID到设备的映射表

```kotlin
// 当前实现
val deviceEntry = availableDevices.entries.elementAtOrNull(devId - 1)

// 建议改进
private val deviceIdMap = mutableMapOf<Int, String>() // id -> deviceName

// 在sendDeviceListUpdate时建立映射
var id = 1
for ((deviceName, device) in availableDevices) {
    deviceIdMap[id] = deviceName
    id++
}

// 在handleImportDevice时使用
val deviceName = deviceIdMap[devId]
```

#### 4. 缺少超时处理
**位置**: UsbService.kt ClientHandler.run()  
**问题**: socket.read()没有设置超时，可能永久阻塞  
**建议**: 设置socket timeout

```kotlin
socket.soTimeout = 30000 // 30秒超时
```

#### 5. 资源泄漏风险
**位置**: UsbService.kt handleImportDevice()  
**问题**: 如果异常发生，importedDevices可能包含不完整状态  
**建议**: 使用try-finally确保状态一致性

### 🟢 轻微问题

#### 6. 日志级别混用
**位置**: UsbService.kt多处  
**问题**: 有些使用Log.d，有些使用log()方法  
**建议**: 统一使用log()方法以便广播

#### 7. 硬编码常量
**位置**: win_gui.py多处  
**问题**: 命令码硬编码为数字（0x1003等）  
**建议**: 定义为常量

```python
CMD_IMPORT_DEVICE = 0x1003
CMD_IMPORT_RES = 0x0004
```

#### 8. 错误消息不够友好
**位置**: win_gui.py handle_import_response()  
**问题**: 技术性的错误码对用户不友好  
**建议**: 提供友好的错误消息映射
