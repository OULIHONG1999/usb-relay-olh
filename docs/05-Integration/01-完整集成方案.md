# USB Relay 项目 - 完整集成方案

## 🎯 最终目标

将 Android 设备连接的 USB 设备通过网络转发给 Windows 电脑，使 Windows 应用能像使用本地设备一样使用远程 USB 设备。

## 🔍 研究结论

### 最佳方案：采用成熟开源项目

**不重新造轮子！**

| 组件 | 选用项目 | 理由 |
|-----|--------|------|
| **Windows 客户端** | usbip-win2 | WHLK 认证驱动，完整 USB/IP 协议，生产级质量 |
| **Android 服务器** | Android-Usbipdcpp | 完整 USB/IP 实现，Kotlin + C++，直接可用 |

### 优势

✅ **零驱动开发** - usbip-win2 已提供驱动
✅ **完整协议** - 标准 USB/IP，生态完善
✅ **快速验证** - 可立即测试
✅ **维护成本低** - 依赖成熟社区维护

## 🔄 集成策略

### 策略 A: 直接使用现有项目 (推荐)

```
usb-relay-olh/
├── Android-Usbipdcpp/  (直接使用)
└── usbip-win2/        (直接使用)
```

**优点**:
- 无需开发，立即可用
- 最稳定可靠

**缺点**:
- 不是在原始 relay-host/relay-client 框架下

### 策略 B: 移植到原始框架

```
usb-relay-olh/
├── relay-client/  (替换为 usbip-win2 客户端)
└── relay-host/    (移植 Android-Usbipdcpp 核心)
```

**优点**:
- 保持原始项目结构
- 可定制化开发

**缺点**:
- 工作量大
- 需要深入理解代码

## 🚀 快速验证指南

### 1. 准备工作

#### 硬件要求
- Android 手机/平板 (Android 9.0+, minSdk 28)
- USB OTG 线
- USB 设备 (U盘、摄像头、打印机等)
- Windows 电脑 (Windows 10/11)
- 同一 WiFi 网络

#### 软件要求
- Android Studio (如需编译 Android 项目)
- Windows 管理员权限 (安装驱动需要)

---

### 2. Windows 端设置 (usbip-win2)

#### 步骤 1: 下载安装包
- 访问: https://github.com/vadimgrn/usbip-win2/releases
- 下载最新版本的 `USBip-Setup-*.exe`

#### 步骤 2: 安装驱动 (需要管理员权限)
⚠️ **重要**: 先创建系统还原点！
```powershell
# 以管理员身份打开 PowerShell
# 运行安装程序
USBip-Setup-*.exe
```

#### 步骤 3: 验证安装
安装完成后，在命令行中测试：
```powershell
usbip --help
```

---

### 3. Android 端设置 (Android-Usbipdcpp)

#### 选项 A: 直接使用 APK (推荐)
1. 访问原项目 GitHub
2. 下载预编译 APK (如果有)
3. 安装到 Android 设备

#### 选项 B: 从源代码编译
1. 打开 `Android-Usbipdcpp/` 项目
2. 用 Android Studio 打开
3. 同步 Gradle 依赖
4. 连接 Android 设备 (开发者选项 + USB 调试)
5. 点击 Run 按钮安装

---

### 4. 完整通信测试

#### Android 端 (服务器)
1. 打开 Android 应用
2. 连接 USB 设备到 Android (通过 OTG 线)
3. 授予 USB 权限
4. 绑定 USB 设备
5. 启动服务器 (默认端口 3240)
6. 查看 Android 设备的 IP 地址

#### Windows 端 (客户端)
1. 打开命令行
2. 列出远程设备：
   ```powershell
   usbip list -r <Android_IP>
   ```
3. 附加远程设备：
   ```powershell
   usbip attach -r <Android_IP> -b <BUS_ID>
   ```
4. 检查 Windows 设备管理器，应该能看到 USB 设备！

---

### 5. 结束使用

#### Windows 端
```powershell
# 列出附加的设备
usbip port

# 分离设备
usbip detach -p <PORT_NUMBER>
```

#### Android 端
1. 在应用中停止服务器
2. 解绑 USB 设备
