# USB Relay 动态设备发现和日志广播 - 问题分析与修复方案

## 当前问题

从你的日志可以看出：

```
[07:03:29] [DEBUG] 收到原始数据 (18 bytes): 48 65 6c 6c 6f 20 66 72 6f 6d 20 73 65 72 76 65 72 21...
[07:03:29] [DEBUG] 收到文本数据: Hello from server!...
```

**问题根源**：
1. **没有使用协议**：服务器发送的是纯文本"Hello from server!"，而不是符合USBIP-OLH协议的格式
2. **没有设备广播**：显示"Sent updated device list with 0 devices"，因为availableDevices为空
3. **日志只是回显**：客户端收到的"Received 14 bytes from client"是服务器简单回显，不是真正的日志推送

## 正确的协议格式

### 协议头部（16字节）
```
Offset  Size  Field       Description
------  ----  ----------  ---------------------------
0       2     command     命令码 (CMD_xxx)
2       4     seq_num     序列号
6       2     dev_id      设备ID (0xFFFF = 全局)
8       4     length      payload长度
12      4     reserved    保留字段
```

### 命令码定义
```c
#define CMD_DEVLIST_REQ     0x0001  // 客户端请求设备列表
#define CMD_DEVLIST_RES     0x0002  // 服务器响应设备列表
#define CMD_KEEPALIVE       0x000A  // Ping
#define CMD_LOG             0x1001  // 日志消息
#define CMD_DEVICE_UPDATE   0x1002  // 设备更新通知
```

## 需要修复的内容

### 1. Android服务端 (UsbService.kt)

#### 当前错误实现：
```kotlin
// ❌ 错误：直接发送文本
output.write("Hello from server!".toByteArray())
output.write(jsonBytes)
```

#### 正确实现：
```kotlin
// ✅ 正确：使用协议格式
private fun sendProtocolMessage(command: Int, devId: Int = 0xFFFF, payload: ByteArray) {
    val out = output ?: return
    val currentSeq = seqNum++
    
    // 写入头部（大端序）
    writeU16BE(out, command)           // 命令码
    writeU32BE(out, currentSeq.toLong()) // 序列号
    writeU16BE(out, devId)              // 设备ID
    writeU32BE(out, payload.size.toLong()) // payload长度
    writeU32BE(out, 0)                  // reserved
    
    // 写入payload
    if (payload.isNotEmpty()) {
        out.write(payload)
    }
    out.flush()
}

// 发送设备更新
private fun sendDeviceListUpdate() {
    val jsonBytes = buildDeviceJson().toByteArray(Charsets.UTF_8)
    sendProtocolMessage(0x1002, 0xFFFF, jsonBytes) // CMD_DEVICE_UPDATE
}

// 发送日志
private fun sendLog(level: Int, message: String) {
    val timestamp = System.currentTimeMillis()
    val msgBytes = message.toByteArray(Charsets.UTF_8)
    
    // 构建payload: timestamp(8) + level(1) + msg_len(4) + message
    val payload = ByteArray(8 + 1 + 4 + msgBytes.size)
    // ... 填充payload ...
    
    sendProtocolMessage(0x1001, 0xFFFF, payload) // CMD_LOG
}
```

### 2. Windows GUI客户端 (win_gui.py)

#### 当前问题：
```python
# ❌ 只是尝试解析JSON，没有解析协议头
def receive_loop(self):
    data = self.sock.recv(8192)
    # 直接尝试找JSON...
```

#### 正确实现：
```python
# ✅ 先解析协议头，再处理payload
def receive_loop(self):
    buffer = b""
    
    while self.running:
        # 读取至少16字节头部
        while len(buffer) < 16:
            data = self.sock.recv(4096)
            if not data:
                return
            buffer += data
        
        # 解析头部
        cmd, seq, dev_id, length, _ = struct.unpack('!HIHHI', buffer[:16])
        buffer = buffer[16:]
        
        # 读取完整payload
        while len(buffer) < length:
            data = self.sock.recv(4096)
            if not data:
                return
            buffer += data
        
        payload = buffer[:length]
        buffer = buffer[length:]
        
        # 根据命令码处理
        if cmd == 0x1002:  # CMD_DEVICE_UPDATE
            self.handle_device_update(payload)
        elif cmd == 0x1001:  # CMD_LOG
            self.handle_log_message(payload)
        elif cmd == 0x000B:  # CMD_PONG
            self.log("Pong received")
```

### 3. 设备绑定时触发广播

在`bindDevice()`方法中：
```kotlin
fun bindDevice(usbManager: UsbManager, device: UsbDevice): DeviceBindResult {
    log("Bind device: ${device.deviceName}")
    
    // 添加到可用设备列表
    availableDevices[device.deviceName] = device
    
    // ✅ 向所有客户端广播设备更新
    broadcastDeviceUpdate()
    
    return DeviceBindResult.Success("device-${availableDevices.size}")
}

private fun broadcastDeviceUpdate() {
    clients.forEach { client ->
        client.sendDeviceListUpdate()
    }
}
```
