# USB Relay 动态设备发现功能测试指南

## 最新更新 (2026-05-14)

### 新增功能
1. **Android端实时设备广播**: 当USB设备插入或移除时，服务器会自动向所有连接的客户端推送更新
2. **Windows客户端动态接收**: 客户端能够实时接收并显示可用的USB设备列表
3. **交互式命令行界面**: 支持查看、选择和连接远程USB设备

## 构建和安装

### Android应用
```bash
cd relay-host/USBRelay
.\gradlew.bat assembleDebug
adb install -r app\build\outputs\apk\debug\app-debug.apk
```

### Windows客户端
```bash
cd relay-client/windows
mkdir build && cd build
cmake .. -G "Visual Studio 17 2022" -A x64
cmake --build . --config Release
```

编译后的可执行文件位于: `relay-client/windows/build/Release/relay-client.exe`

## 测试步骤

### 1. 启动Android服务端
1. 在Android设备上打开"USB Relay"应用
2. 点击"Start Server"按钮启动服务
3. 记下显示的IP地址（例如: 192.168.1.100）

### 2. 连接USB设备到Android
1. 通过OTG线将USB设备连接到Android设备
2. 应用会自动检测设备并显示在列表中
3. 授予USB权限（如果提示）

### 3. 启动Windows客户端
```bash
cd relay-client/windows/build/Release
.\relay-client.exe <android-ip-address> 3240
```

例如:
```bash
.\relay-client.exe 192.168.1.100 3240
```

### 4. 使用客户端命令

连接成功后，可以使用以下命令：

- **list**: 显示当前可用的USB设备列表
  ```
  relay-client> list
  
  Remote USB Devices (2):
    ID   VID      PID      Class
    ---- -------- -------- ------
    1    0x0403   0x6001   0x00
    2    0x1a86   0x7523   0xff
  ```

- **import <id>**: 导入指定ID的USB设备（功能待完善）
  ```
  relay-client> import 1
  Importing device 1...
  Device import feature not fully implemented yet.
  ```

- **refresh**: 刷新设备列表
  ```
  relay-client> refresh
  Refreshing device list...
  Device list refreshed.
  ```

- **quit**: 退出客户端
  ```
  relay-client> quit
  ```

### 5. 测试动态设备发现

1. 保持Windows客户端运行
2. 在Android设备上插入新的USB设备
3. 观察Windows客户端是否自动收到设备更新通知
4. 使用`list`命令查看新设备

## 协议扩展

新增了以下命令码（见 `relay-client/common/protocol.h`）：

- `CMD_DEVICE_UPDATE (0x1002)`: 服务器主动推送设备列表更新
- `CMD_IMPORT_DEVICE (0x1003)`: 客户端请求导入特定设备

## 已知限制

1. **设备导入功能未完成**: 目前`import`命令只是占位符，需要实现真正的USB设备虚拟化
2. **JSON解析简化**: 设备列表以简单JSON格式传输，但未实现完整的JSON解析器
3. **无图形界面**: Windows客户端目前只有命令行界面

## 下一步开发

1. 实现完整的USB设备导入和虚拟化功能
2. 添加JSON解析库（如cJSON或json-c）
3. 为Windows客户端开发图形用户界面
4. 添加认证和加密机制
5. 支持更多USB设备类型（HID、音频、视频等）

## 故障排除

### Android端问题
- 确保USB OTG功能正常
- 检查USB权限是否已授予
- 查看应用日志确认设备检测

### Windows端问题
- 确保防火墙允许3240端口通信
- 检查网络连接是否正常
- 确认Android设备IP地址正确

### 连接问题
- 确保Android和Windows在同一网络
- 尝试ping Android设备IP确认连通性
- 检查Android应用是否正在运行服务器
